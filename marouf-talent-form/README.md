# معروف – نموذج دعم المواهب الكروية للأطفال ⚽

موقع نموذج عربي احترافي (RTL) يجمع طلبات الأطفال الموهوبين في كرة القدم،
ويخزّن البيانات تلقائياً، مع لوحة إدارة بسيطة لعرض الطلبات وتصديرها إلى Excel.

> **الفكرة:** «معروف يتبنّى حلمك» — لأنّ كل موهبة تستحق أن تُكتشف، وكل حلم يستحق الدعم.

---

## ✨ المزايا

- نموذج عربي كامل (RTL) بتصميم فاخر مستوحى من القهوة (ذهبي / بنّي / بيج) وخط **Cairo**.
- تصميم متجاوب يعمل على الجوال والحاسوب.
- تحقّق من الحقول المطلوبة في الواجهة والخادم.
- رسالة نجاح عربية: «شكراً لك، تم استلام بياناتك بنجاح.»
- تخزين تلقائي مع تسجيل وقت كل طلب (SQLite).
- لوحة إدارة محميّة بكلمة مرور: جدول بالطلبات + زر **تحميل Excel (.xlsx)**.
- دعم اختياري لمزامنة البيانات مع **Google Sheets**.

## 🚀 التشغيل محلياً

المتطلّب: **Python 3.10+**

```bash
# 1) ادخل مجلد المشروع
cd marouf-talent-form

# 2) أنشئ بيئة افتراضية وفعّلها
python -m venv venv
source venv/bin/activate        # على ويندوز: venv\Scripts\activate

# 3) ثبّت المكتبات
pip install -r requirements.txt

# 4) شغّل التطبيق
python app.py
```

ثم افتح المتصفح على: **http://localhost:5000**

| الصفحة | الرابط |
|--------|--------|
| النموذج | `/` |
| لوحة الإدارة | `/admin` |
| تحميل Excel | `/admin/download` |

- ادخل على `/admin` فتُطلب كلمة المرور.
- كلمة المرور الافتراضية: `marouf2025`
- **غيّرها** عبر متغيّر البيئة `ADMIN_PASSWORD` (مهم قبل النشر).

```bash
export SECRET_KEY="نص_عشوائي_طويل"
```

## ☁️ النشر على Render (رابط عام واحد)

1. ارفع المشروع إلى مستودع على **GitHub**.
2. ادخل [render.com](https://render.com) → **New** → **Web Service** → اربط المستودع.
3. الإعدادات:
   - **Runtime:** Python
   - **Build Command:** `pip install -r requirements.txt`
   - **Start Command:** `gunicorn app:app`
4. في تبويب **Environment** أضف:
   - `ADMIN_PASSWORD` = كلمة مرور الإدارة
   - `SECRET_KEY` = نص عشوائي طويل
5. اضغط **Create Web Service**. ستحصل على رابط عام مثل:
   `https://marouf-talent-form.onrender.com`

> يوجد ملف `render.yaml` جاهز إن أردت النشر بضغطة واحدة عبر **Blueprint**.

### ⚠️ ملاحظة مهمة عن حفظ البيانات على الخطة المجانية
قرص Render في الخطة المجانية **مؤقت**؛ أي قد تُمسح قاعدة بيانات SQLite عند إعادة تشغيل الخدمة.
للحفظ الدائم اختر أحد الحلول:
- **الأسهل:** فعّل **Google Sheets** (الخطوات بالأسفل) — تبقى كل البيانات في جدولك.
- **بديل:** أضف **Persistent Disk** في Render واربطه بمجلد `data/` (خطة مدفوعة).

## 📊 تفعيل Google Sheets (اختياري)

عند تفعيله، يُضاف كل طلب إلى Google Sheet تلقائياً (مع بقاء الحفظ المحلي).

1. ادخل [Google Cloud Console](https://console.cloud.google.com) وأنشئ مشروعاً.
2. فعّل **Google Sheets API**.
3. أنشئ **Service Account** ثم أنشئ مفتاح **JSON** ونزّله باسم `credentials.json` بجانب `app.py`.
4. أنشئ Google Sheet جديداً، وانسخ معرّفه من الرابط (الجزء بين `/d/` و `/edit`).
5. **شارك** الجدول مع البريد الإلكتروني الخاص بالـ Service Account (صلاحية Editor).
6. ثبّت مكتبات Google (موجودة في `requirements.txt`):
   ```bash
   pip install gspread google-auth
   ```
7. اضبط متغيّرات البيئة:
   ```bash
   export GOOGLE_SHEET_ID="معرّف_الجدول"
   export GOOGLE_CREDS_FILE="credentials.json"
   ```
   على Render: ارفع `credentials.json` كـ **Secret File** وأضف `GOOGLE_SHEET_ID` في Environment.

إن لم تُضبط هذه المتغيّرات، يعمل المشروع طبيعياً بالتخزين المحلي فقط.
